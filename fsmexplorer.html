Finite State Machine Explorer Help

About

Finite State Machine (FSM) Explorer is an educational tool for exploring FSM controllers, the type of FSM used when designing digital systems. The tool allows the user to draw an FSM diagram, specify node and arc parameters, and functionally simulate the resulting FSM. Mealy and Moore output styles are supported and can be mixed within one FSM, as long as the result is a valid fsm.Input expressions can be Boolean expressions (e.g. a+c'), binary strings (e.g. 101), or can use cube notation "1--". Output expressions can be binary strings (e.g. 101), assignment statements (e.g. p=1,q=0), or assertions (e.g. p,r).

Basic Design Rules

An FSM diagram is drawn by placing nodes, interconnecting them by arcs, and specifying arc conditions and node and arc output values. A single reset arc is allowed per FSM, and simulation requires reset be asserted to set the initial state. For and nodes Si and Sj, there can be only a single arc from Si to Sj, and from Sj to Si (including the self arcs from a node to itself). This is not a limitation, as multiple input/output conditions (parallel arcs) can be specified for each arc (e.g. a'/0, a/1).

An arc with no condition, or one marked with input "-", is considered always taken. Except when using the assertion output style, where outputs are assumed deasserted unless explicitly asserted, default values are not assumed for outputs. There is no method for making an arc as default in case no other arc matches; such usage is not standard.

When nodes are placed, the tool will automatically assign the names S0, S1, etc. If the user changes the first node name to SA, the tool will automaticall switch to naming convention SA, SB, etc. If other custom state names are assigned, the tool will cease automatic naming of states.

Just as in real world designs, all arc conditions leaving a node must be exhaustive and mutually exclusive such that there is always exactly one next state. During simulation, the output 'X' indicates there are at least two next states specified and the output 'U' indicates there is no next state specified.

The tool is capable of inferring input/output names and the meaning of input expressions using several different styles. In some cases, however, this is not possible. For example, the Boolean expression a+bc could refer to inputs "a", "b", and "c" or inputs "a" and "bc". The tool will assume the latter, and the user can force meaning by separating b and c: a+b c, a+(b c), a+b&c, a+(b)(c), a+b.c, etc. If the user explicitly identifies the input variable names as a, b, and c, then expressions such as a+bc are fine.

In a single FSM, a single input and output style should be used or the simulation will generally not show any outputs at all. In most cases, mixing styles makes the meaning ambiguous. For example, if one arc uses "11/0" and another uses a'+c/0, it is not possible to tell which literal in the first expression is a and which is c. Similar problems can arise with mixing output styles.

FSM Editor Tools

FSM diagrams are drawn using two primary tools:

Node Tool
Allows the user to click on an empty place on the canvas and place a new node. Double-clicking a node brings up a property editor that allows the user to edit the state name and optional Moore outputs. Nodes can be repositioned using click-and-drag.
Arc Tool
Allows the user to create a reset arc or arcs between nodes. Click on an empty spot on the canvas and drag the arc to a node to create a reset node. Click on an existing node to create a self arc. Click on an existing node and drag the arc to another to create an arc between nodes. Double-clicking an arc brings up a property editor that allows the user to edit the arc condition and optional Mealy outputs. Arcs can be reshaped by click-and-drag.
Select Tool
Allows the user to select individual or groups of arcs and nodes to reposition or delete them. Clicking on an empty part of the canvas unselects all elements. Individual elements are selected by clicking them. Mutliple elements are selected by dragging the selection rectangle. Once a multiple selection is made, additional elements can be added by clicking on them (it is not possible to deselect individual elements). Selected elements may be moved, deleted using the delete button, or auto-arranged.
Arrange Tool
Attempts to arrange selected nodes on a circle, and position their arcs in a neatly organized pattern. For best results, roughly position the nodes how you want them (triangle, circle, square, etc.) before arranging.

FSM Input and Output Styles

Input conditions on arcs and outputs on arcs (Mealy) and nodes (Moore) allow flexiblty in terms of how they are specified, but certain rules apply for the tool to be able to infer meaning of the typed expressions. It is generally not a good idea to mix input styles in one FSM. It is not possible to mix output styles in one FSM (the FSM outputs will be ignored in such cases). Any arc with no input condition, or with a single -, is considered always taken and should be the only outgoing arc of the state.

FSM Input Styles

Binary Input Style
Input(s) are specified as binary strings. The order and number of inputs must be the same on every arc and/or node. The inputs are automatically named a[0], a[1], etc. If user specified names are provided (by double-clicking on the FSM canvas), they will be assigned to the binary input from left-to-right. Don't cares are possible using -, ? or X.
Boolean Expression Input Style
Input(s) are specified as Boolean expressions like !a|b , a'+b , a(b+c') , etc. The number and names of the inputs are inferred from the expressions. Variable names are flexible but generally best kept short. Because it is not possible to determine whether expressions like ab' refer to one variable called ab or two variables a and b, it is necessary to use an AND symbol between such literals if the meaning is a&b. Valid choices are &,*,. or using parenthesis ( e.g. (a)(b)). All of the following, therefore, represent A and B: A B, A(B), A*B, A.B, and A&B. NOT can be specified using ' after a literal or parenthesis, or ! before a literal or parenthesis. OR uses the symbols + or | or ||.

FSM Output Styles

Warning: mixing output styles within the same FSM will cause the simulator to not show any outputs.

Positional (Binary) Style
Specifies output(s) as binary strings. The order and number of outputs must be the same on every arc and/or node. The outputs are automatically named f[0], f[1], etc. If user specified names are provided (by double-clicking on the FSM canvas), they will be assigned to the binary input from left-to-right.
Explicit Output Assignment Style
Specifies oututs using lists of explicit signal assignments, such as p=0, q=1, etc. There is no default value assumed, so assignments for every output must be made for every arc in a Mealy FSM orevery state in a Moore.
Implicit Output Assignment Style
Specifies outputs using lists of signals that should be asserted (made high). For example, "p q r" means to make p, q and r all high, whereas "p,q" means make only p and q high. Asserting active low signals is not possible. When not specified, outputs are assumed to be 0. This style is often used in controllers that output control signals to control a datapath.

Simulation

Starting a Simulation

Simulate the current FSM by clicking the Simulate Button. To edit the FSM again, use the Edit button when in simulation mode.Note that edited waveforms are not saved when switching to Edit mode.

Editing Input Waveforms

Input waveforms may be edited. Click on a waveform to insert a half-cycle pulse (negative or positive) at the click point. Click and drage to insert a longer pulse. Click and drag a signal edge to make a pulse wider or narrower. Click on a signal edge to delete it.

Using the Timing Marker

The timing marker is useful to help read the timing diagrams. The current state of the FSM, along with the active arc, is shown for the marker position. Click on the square at the bottom to drag it around. Click on the clock signal to align it with a clock edge. Click and drag across the clock signal to follow the FSM. The marker will "snap" just prior to and after an active edge to show state transitions.

Troubleshooting

Output Waveforms don't appear
usually means there is an error in your arc or node outputs or you mixed output styles (See above).
Output Waveforms have red X's
usually means your arc conditions are not mutually exclusive.
Output Waveforms have red U's
usually means
  • a forgotten the reset arc
  • forgetting to assert reset in the timing diagram
  • forgetting to deassert reset in the timing diagram
  • arc conditions are not mutually exclusive; two or more outgoing arcs have overlapping Boolean expressions.

Finite State Machine Explorer

About

Finite State Machine (FSM) Explorer is an educational tool for exploring FSM controllers, the type of FSM used by engineers when designing digital systems. The tool allows the user to draw an FSM diagram, specify node and arc parameters, and functionally simulate the resulting FSM.

Features

FSM Editor

  • Unlimited history with undo/redo
  • File save and load
  • Export FSM image
  • Autoarrange to help create nice looking diagrams for export
  • Automatic or manual state and input/output signal naming
  • Support for Mealy and Moore outputs
  • Support for several styles of specifying arc inputs and FSM outputs

FSM Simulator

  • Accurate functional simulation including multiply-driven (X) and unknown (U) outputs
  • Export timing diagram as image
  • Timing pointer to help read diagram
  • Timing pointer snap to clock edges with before and after active edge snapping
  • Graphical input waveform editor

Getting Started

The easiest way to get started is to click Help and select one of the example FSMs to play around with. The Help section will also provide some information on using the drawing and simulation tools.
